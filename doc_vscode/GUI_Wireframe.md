## 2. コンポーネント詳細

### 2.1 ヘッダー（Header）

```
┌─────────────────────────────────────────┐
│  Weekly Kanban - Week 27/2026    [×]   │
├─────────────────────────────────────────┤
│  Created: 2026-07-02 | Updated: 15:30   │
└─────────────────────────────────────────┘
```

**含まれる要素**:
- Title（プロジェクト名 + 週情報）
- Close ボタン（Webview パネルを閉じる）
- メタデータ（作成日、更新日）
- Refresh ボタン（オプション）

---

### 2.2 カラムヘッダー（Column Header）

```
┌──────────────────────────┐
│ 📋 TO DO                 │
│ (3 items)                │
├──────────────────────────┤
│ [Filter] [Sort] [▼ More] │
└──────────────────────────┘
```

**含まれる要素**:
- カラム名（アイコン + テキスト）
- アイテム数表示
- フィルター/ソート ボタン（簡易版、Phase 2+）
- ドロップダウンメニュー

---

### 2.3 Kanban カード（Card）

```
┌──────────────────────────┐
│ [  ] Task 1              │  ← チェックボックス + タイトル
├──────────────────────────┤
│ ID: task-1               │  ← メタデータ
│ Label: feature, urgent   │
│ Due: 2026-07-10          │
├──────────────────────────┤
│ ⚙ ⋮ [Priority] [×]     │  ← アクションボタン（右ホバー時のみ表示）
└──────────────────────────┘
```

**カード内容**:
- **チェックボックス + タイトル**: ワンクリックで完了状態切り替え可
- **メタデータ行**: ID, Label (badges), Due date
- **アクション行**: 編集, メニュー, 削除（ホバー時のみ）

**ドラッグ動作**:
- マウスダウン → カード背景を薄く（ドラッグ開始表示）
- ドラッグ中 → ゴーストイメージ + ドロップターゲット強調
- ドロップ → ターゲットカラムへアニメーション移動

---

### 2.4 "+ Add Card" ボタン

```
┌──────────────────────────┐
│ [+ Add Card]             │
└──────────────────────────┘
```

**機能**（Phase 2+）:
- クリック → インライン入力モード
- 入力フィールド表示
- Save/Cancel ボタン

**MVP では**: 無効化（コメント化）

---

### 2.5 ステータスバー（Footer）

```
┌─────────────────────────────────────────┐
│ Status: Synced ✓ (Last update: 15:30)   │
└─────────────────────────────────────────┘
```

**状態表示**:
- ✓ **Synced**: 最後の同期が成功
- ⏳ **Syncing**: 同期中（スピナーアニメーション）
- ✗ **Error**: 同期失敗（エラーメッセージ表示）

## 3. インタラクション フロー

### 3.1 カード移動（Drag & Drop）

```
1. ユーザーが Task 1 をマウスダウン
   ↓
2. Card visual changed: opacity 0.7, cursor: grabbing
   ↓
3. ユーザーが DOING 列へドラッグ
   ↓
4. Drop zone highlighted: green border, background tint
   ↓
5. ユーザーがドロップ
   ↓
6. Animation: Card slides to DOING column
   ↓
7. Webview sends: { action: 'move', cardId: 'task-1', targetColumn: 'Doing' }
   ↓
8. Extension queues: Markdown update (debounced 300ms)
   ↓
9. User saves (Ctrl+S)
   ↓
10. Extension: Apply changeset → Markdown updated
    ↓
11. Markdown save detected → Kanban re-renders
    ↓
12. Status: "Synced ✓" displayed
```

---

### 3.2 カード完了状態の切り替え

```
1. ユーザーが Task 1 チェックボックスをクリック
   ↓
2. Visual: Checkbox checked [x], card opacity 0.8
   ↓
3. Card moves to DONE column
   ↓
4. Webview sends: { action: 'complete', cardId: 'task-1' }
   ↓
5-12. [同上：save → sync → status]
```

## 6. アクセシビリティ

### 6.1 キーボード操作

| 操作 | キー | 動作 |
|------|------|------|
| カード選択 | Tab | 次のカードへフォーカス移動 |
| カード移動 | Arrow キー | 列間でカード移動 |
| 完了切り替え | Enter | チェックボックス切り替え |
| カード削除 | Delete | カード削除（確認ダイアログ） |

### 6.2 スクリーンリーダー対応

- `role="main"` - メインコンテンツ領域
- `role="region" aria-label="column"` - カラムのセマンティック
- `aria-label` - アイコンボタンの説明
- `aria-live="polite"` - ステータス更新通知

## 7. デザイン決定

### 決定 1: 列固定（カスタマイズ不可、MVP）

**理由**: 複雑さ減少、テスト容易化

**代替案**: Phase 2+で列のカスタム名対応

---

### 決定 2: インラインメタデータ表示

**理由**: カード外観を保ちながら必須情報を表示

**代替案**: ポップアップ詳細ビュー（Phase 2+）

---

### 決定 3: ドラッグ直後に自動保存（してない）

**理由**: ユーザーが保存意思を明示 → データ安全性向上

**デメリット**: ユーザーが保存忘れのリスク

**対策**: ステータスバーに "Unsaved changes" 表示予定（Phase 2）
